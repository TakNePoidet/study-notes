Развёрнутый ответ

Что такое JavaScript

JavaScript — это многопарадигменный язык программирования, изначально созданный Бренданом Эйхом в 1995 году для оживления страниц в браузере Netscape. Назвали его так из маркетинговых соображений — в попытке прицепиться к популярности Java, хотя общего у языков мало.

JavaScript — динамически типизированный, интерпретируемый (точнее, JIT-компилируемый), поддерживает функции как объекты первого класса, замыкания, прототипное наследование (с ES2015 — и привычный классовый синтаксис). Стандарт языка называется ECMAScript, и каждый год выходит новая версия — ES2015, ES2016, …, ES2024. Современный JS — это ES2015+ с модулями (import/export), let/const, стрелочными функциями, классами, деструктуризацией, async/await.

Где применяется JavaScript

JS перестал быть «языком для оживления страниц» — сейчас это один из самых универсальных языков.

Фронтенд остаётся главной сферой: интерактивные интерфейсы, SPA на Vue, React, Svelte, Angular, Solid.

Бэкенд. Появление Node.js (2009) сделало JS серверным языком. Сейчас активно развиваются Deno и Bun. На JS пишут API, реал-тайм-серверы, GraphQL-сервисы, CLI-утилиты.

Мобильная разработка. React Native, Expo и Capacitor позволяют писать кросс-платформенные мобильные приложения на JS/TS.

Десктоп. Electron (VS Code, Slack, Discord) и Tauri используют веб-технологии для десктопных приложений.

Embedded и IoT. Espruino, Johnny-Five — JS на микроконтроллерах.

ML и AI в браузере. TensorFlow.js позволяет запускать модели прямо в браузере.

Скриптование и автоматизация. GitHub Actions, Cypress, Playwright, серверлесс-функции в AWS Lambda, Cloudflare Workers — везде JS.

Этот универсализм делает JavaScript практически обязательным навыком для веб-разработчика.

Что такое DOM

DOM (Document Object Model) — это программная модель HTML-документа, построенная браузером после парсинга. Каждый элемент HTML становится узлом в дереве, и JavaScript может его читать и изменять.

Например, документ:

<html>
  <head><title>Page</title></head>
  <body>
    <h1>Hello</h1>
    <p>Lorem ipsum</p>
  </body>
</html>

После парсинга в DOM:

Document
└── html
    ├── head
    │   └── title — "Page"
    └── body
        ├── h1 — "Hello"
        └── p — "Lorem ipsum"

JS получает доступ к этому дереву через глобальные объекты document и window. Каждый узел — это объект с набором свойств и методов, и через них можно манипулировать страницей.

Базовые операции с DOM

Поиск элементов.

const root  = document.querySelector('#app');         // первый по селектору
const items = document.querySelectorAll('.item');     // все по селектору
const byId  = document.getElementById('user');        // быстрый поиск по id

querySelector принимает CSS-селектор, что очень удобно: можно искать .class, #id, tag, сложные комбинации.

Создание и вставка элементов.

const li = document.createElement('li');
li.textContent = 'Новый пункт';
li.classList.add('item');
root.appendChild(li);

Дополнительно есть insertBefore, insertAdjacentHTML, replaceChildren — для разных способов вставки.

Изменение атрибутов и стилей.

li.dataset.userId = 42;                  // data-user-id="42"
li.setAttribute('aria-current', 'true'); // обычные атрибуты
li.style.color = 'crimson';              // инлайновый стиль
li.classList.toggle('active');           // переключение класса

classList — рекомендуемый способ работы с классами; он удобнее, чем строковая манипуляция className.

Удаление.

li.remove();              // современный
li.parentNode.removeChild(li);  // старый эквивалент

События

Через DOM можно подписываться на события и реагировать на действия пользователя.

document.querySelector('#btn').addEventListener('click', (e) => {
  e.preventDefault();
  console.log('clicked', e.target);
});

Каждое событие проходит три фазы: capture (от корня вниз к цели), target (на самом элементе), bubble (вверх обратно к корню). По умолчанию обработчик срабатывает на фазе bubble; третий аргумент addEventListener позволяет перехватить событие на фазе capture.

Делегирование событий — мощный паттерн: вместо того чтобы вешать обработчик на каждый элемент списка, вешаем один на родителя и определяем цель внутри. Это работает за счёт всплытия.

document.querySelector('#list').addEventListener('click', (e) => {
  const item = e.target.closest('.item');
  if (item) item.classList.toggle('done');
});

Производительность и современные API

Прямые манипуляции с DOM дорогие, потому что вызывают reflow (пересчёт компоновки) и repaint (перерисовку). Несколько правил для производительной работы:

  • Группировать изменения. Если нужно вставить много элементов — использовать DocumentFragment или строить HTML в строке и вставлять одним вызовом.
  • Не читать layout-свойства (offsetTop, offsetWidth) сразу после записи — это вызывает forced reflow.
  • Тяжёлые операции — внутри requestAnimationFrame, чтобы синхронизироваться с кадрами браузера.

Современные API расширяют возможности работы с DOM:

  • MutationObserver — реакция на изменения DOM (новые узлы, удалённые, изменения атрибутов).
  • IntersectionObserver — узнать, что элемент виден на экране (используется для lazy-load изображений и бесконечной прокрутки).
  • ResizeObserver — отслеживать изменение размеров элементов.
  • Shadow DOM — изолированный DOM для Web Components, не пересекающийся со стилями страницы.

Виртуальный DOM и фреймворки

В реальных приложениях прямые манипуляции DOM редко пишут руками — это делают фреймворки: React, Vue, Svelte. Они используют концепцию виртуального DOM (или похожих идей вроде Svelte-компилятора): мы описываем, как должен выглядеть интерфейс, а фреймворк сам считает разницу между предыдущим и новым состоянием и применяет минимально необходимые операции к реальному DOM. Это и удобнее, и обычно быстрее, чем писать вручную.

Что важно сказать в итоге

JavaScript — это динамически типизированный, многопарадигменный язык, на котором держится современный веб и не только: фронтенд, серверная разработка, мобильные, десктоп, IoT. DOM — это объектное представление HTML-страницы, через которое JS читает и изменяет интерфейс. Базовые операции — поиск, создание, изменение, удаление узлов и работа с событиями. На практике прямые манипуляции DOM скрыты за фреймворками, но понимание того, как всё устроено внизу, остаётся обязательным для разработчика.
