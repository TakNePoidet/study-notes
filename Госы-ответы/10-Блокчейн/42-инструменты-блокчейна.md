## Развёрнутый ответ

### Базовый стек

Для разработки смарт-контрактов на Ethereum нужно несколько инструментов разных уровней:

- **Geth** — клиент Ethereum-узла, на котором можно поднять свой блокчейн или подключиться к существующему.
- **Git** — версионный контроль, без него любая команда работает плохо.
- **Редактор кода** — VS Code, Sublime Text, IntelliJ с плагинами Solidity.
- **Remix IDE** — браузерная среда для быстрого прототипирования.
- **Ganache** — локальный EVM-блокчейн для тестов.
- **Hardhat / Foundry** — современные фреймворки для серьёзной разработки.
- **MetaMask** — кошелёк-расширение для подключения dApp.

Это «классический» учебный набор. Современная коммерческая разработка ушла от Ganache/Remix к Hardhat/Foundry, но базовые инструменты остаются востребованы.

### Geth (go-ethereum)

**Geth** — это официальная Go-реализация Ethereum-клиента. Им можно подключиться к mainnet, тестовой сети (Sepolia, Holesky) или поднять собственную приватную сеть.

Установка на Ubuntu/Debian:

```bash
sudo add-apt-repository -y ppa:ethereum/ethereum
sudo apt-get update
sudo apt-get install ethereum
geth version
```

Запуск приватной сети (подробнее в вопросе 43):

```bash
geth --datadir ./data init genesis.json
geth --datadir ./data --networkid 12345 \
     --http --http.api eth,net,web3,personal,admin
```

Подключение в консоли:

```bash
geth attach http://127.0.0.1:8545
```

Что можно делать в консоли:

```javascript
> eth.accounts
> personal.newAccount("password")
> eth.getBalance(eth.accounts[0])
> eth.sendTransaction({from: eth.accounts[0], to: eth.accounts[1], value: web3.toWei(1, "ether")})
> miner.start(); miner.stop();
```

Geth выступает и как кошелёк, и как сервер для подключения внешних инструментов (Remix, Hardhat, MetaMask) по JSON-RPC.

### Git

Стандартный инструмент управления версиями. Смарт-контракты — это код, и их обязательно нужно держать в Git: история изменений, ревью, теги релизов, синхронизация в команде.

```bash
git clone https://github.com/myorg/dapp.git
git checkout -b feature/staking
git commit -am "Add staking contract"
git push origin feature/staking
```

GitHub, GitLab, Gitea — где это всё хранить. Для смарт-контрактов также важна **публикация открытого кода** на Etherscan для прозрачности (verified source).

### Редакторы кода

**VS Code** — стандарт для большинства разработчиков. Плагины:

- **Solidity** (Juan Blanco) — подсветка, автодополнение, навигация по контрактам.
- **Hardhat for VS Code** — интеграция с Hardhat.
- **Solidity Visual Developer** — диаграммы зависимостей контрактов.

**Sublime Text** — лёгкий редактор, есть пакет Solidity. Используется реже.

**JetBrains IDEA / WebStorm** — плагин Solidity, удобно если уже работаете в JetBrains-экосистеме.

### Remix IDE

**Remix** — это **браузерная** IDE для Solidity по адресу [remix.ethereum.org](https://remix.ethereum.org/). Идеальна для обучения и прототипирования.

Возможности:

- **Встроенный компилятор solc**, выбор версии.
- **Деплой** на любую сеть через MetaMask, локальную **Remix VM** (in-memory EVM), Hardhat Network.
- **Отладчик** с пошаговым выполнением EVM-инструкций.
- **Плагины**: Slither (статический анализ), верификация на Etherscan, генерация тестов.
- **Файловая система** в браузере или подключение GitHub.

Минусы: при больших проектах становится неудобно, нет нормальной CI/CD-интеграции. Поэтому для production обычно используют Hardhat или Foundry.

### Ganache

**Ganache** (Truffle Suite) — локальный EVM-блокчейн с 10 предзаполненными аккаунтами (по 100 ETH каждый). Удобен для быстрого тестирования.

Установка:

```bash
npm install -g ganache
ganache --port 8545 --mnemonic "test test test test test test test test test test test junk"
```

Есть и графическая версия — Ganache UI. Но в сентябре 2023 Truffle Suite (включая Ganache) был **закрыт** компанией ConsenSys. Инструмент ещё работает, но новые проекты обычно выбирают **Anvil** из Foundry или **Hardhat Network**.

### Современная альтернатива — Hardhat и Foundry

**Hardhat** — JavaScript/TypeScript-фреймворк, стандарт де-факто:

```bash
mkdir myproject && cd myproject
npm init -y
npm install --save-dev hardhat
npx hardhat init                   # выбрать "TypeScript project"
npx hardhat compile
npx hardhat test
npx hardhat node                   # локальный узел (аналог Ganache)
npx hardhat run scripts/deploy.ts --network localhost
```

Плюсы: огромная экосистема плагинов, удобная конфигурация, отладчик, форк mainnet для тестов.

**Foundry** — Rust-фреймворк нового поколения:

```bash
curl -L https://foundry.paradigm.xyz | bash
foundryup
forge init myproject
forge build
forge test                         # тесты пишутся на самом Solidity!
anvil                              # локальный узел (быстрее Ganache)
cast send <addr> "transfer(address,uint256)" 0x... 100
```

Плюсы: тесты пишутся на Solidity (нет переключения языков), очень быстрый, мощный fuzz-testing, удобный CLI.

В 2026 году большинство профессиональных команд использует именно Hardhat или Foundry, иногда оба одновременно.

### Кошельки и провайдеры

**MetaMask** — самое популярное расширение-кошелёк для браузера и мобильный. Подключает dApp к Ethereum-сетям, подписывает транзакции, хранит ключи. Альтернативы: Rainbow, Rabby, Trust Wallet, Coinbase Wallet.

**Провайдеры RPC.** Для подключения к публичным сетям без поднятия своего узла используют:

- **Alchemy** — мощный RPC с аналитикой;
- **Infura** (ConsenSys) — классический провайдер;
- **QuickNode**, **Ankr** — альтернативы;
- **Public RPC** — бесплатные, но менее надёжные.

### Тестовые сети

Для отладки используются **тестовые сети** Ethereum:

- **Sepolia** — основная тестовая сеть, monetary aware.
- **Holesky** — для тестов валидации (PoS).
- **Goerli** — устаревшая, deprecated.

Тестовые ETH можно получить через **faucets** (бесплатные раздатчики).

### Полный минимальный стек

В итоге:

```
ОС         : Linux / macOS / Windows
Node.js LTS: для Hardhat / ethers.js
Git        : контроль версий
Editor     : VS Code + Solidity plugin
Local chain: Hardhat Network или Anvil (Foundry); Ganache — legacy
Wallet     : MetaMask
Framework  : Hardhat или Foundry
RPC        : Alchemy / Infura для тестов на Sepolia
Explorer   : Etherscan для просмотра контрактов
```

### Что важно сказать в итоге

Для разработки на Ethereum в учебной программе используется набор инструментов: **Geth** как узел, **Git** для версионирования, редактор кода (Sublime Text, VS Code) с плагинами Solidity, **Remix IDE** для прототипирования и **Ganache** как локальный тестовый блокчейн. Современная коммерческая разработка ушла от Ganache к **Hardhat** и **Foundry** — это более мощные фреймворки с тестированием, конфигурацией, плагинами. Для подключения dApp к сетям применяется **MetaMask**, для публичных RPC — Alchemy, Infura. Тестовая сеть — Sepolia. Этот стек покрывает всё: от первой строчки кода до публикации production-контракта.
