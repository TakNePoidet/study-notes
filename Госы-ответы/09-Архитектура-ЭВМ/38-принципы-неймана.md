## План ответа

1. Архитектура фон Неймана и её принципы.
2. Базовая структура ЭВМ: CPU, память, шины, I/O.

4. Альтернативы: гарвардская архитектура, RISC vs CISC.
5. Устройства ввода-вывода и способы взаимодействия с CPU.
6. Современные тенденции.

## Развёрнутый ответ

### Архитектура фон Неймана

Большинство современных компьютеров построены по **архитектуре фон Неймана**, сформулированной в 1945 году Джоном фон Нейманом в работе «First Draft of a Report on the EDVAC». Это была фундаментальная статья, заложившая основу всей последующей вычислительной техники.

Главные **принципы Неймана**:

1. **Принцип программного управления.** Программа состоит из последовательности команд, которые исполняются процессором одна за другой. Поток управления может быть изменён условными переходами.

2. **Принцип однородности памяти.** Команды и данные хранятся в **одной и той же памяти** и неотличимы по способу хранения — это просто байты. Это даёт огромную гибкость: программа может модифицировать саму себя, компиляторы могут генерировать код в память и исполнять его (JIT-компиляция).

3. **Принцип двоичного представления.** Вся информация — команды, данные, адреса — кодируется в двоичном виде.

4. **Принцип адресности памяти.** Память состоит из ячеек с последовательными адресами. Обращение к данным идёт по адресу.

5. **Принцип условного перехода.** Поток управления зависит от данных — есть ветвления. Это и делает компьютер программируемым.

### Базовая структура ЭВМ

Классическая структура:

```
+-----------+        +-----------+        +----------------+
|  Memory   | <----> |    CPU    | <----> |  I/O Devices   |
+-----------+        +-----------+        +----------------+
                 Шина (Bus): данные / адреса / управление
```

Все компоненты связаны **шинами**: шина данных, шина адресов, шина управления. По шине данных передаются собственно байты, по шине адресов — куда обращаемся, по шине управления — сигналы (читать/писать, прерывание).

### Процессор (CPU)

Процессор — «мозг» компьютера. Его основные части:

- **АЛУ (арифметико-логическое устройство)** — выполняет операции над данными.
- **УУ (устройство управления)** — декодирует команды, управляет всем.
- **Регистры** — самая быстрая память внутри CPU, единицы наносекунд.
- **Кэш L1 / L2 / L3** — иерархия памяти ближе к CPU. L1 — единицы наносекунд, L3 — десятки.
- **MMU (Memory Management Unit)** — аппаратная трансляция виртуальных адресов в физические.

**Конвейер инструкций.** Современный CPU выполняет команды не последовательно, а **конвейерно**: пока одна декодируется, другая загружается, третья исполняется, четвёртая пишется. Это даёт огромный рост производительности.

**Out-of-order execution** — команды выполняются не в порядке программы, а в порядке готовности данных, с последующей переупорядочкой результатов.

**Branch prediction** — предсказание условных переходов: процессор «угадывает», как пойдёт ветвление, и начинает выполнять команды наперёд. Это критично для производительности.

**Спекулятивное выполнение** — следствие branch prediction. Принесло проблемы безопасности: атаки Spectre и Meltdown (2018) эксплуатируют именно его.

**SIMD-инструкции** (SSE, AVX, AVX-512, ARM NEON) — одна команда обрабатывает сразу несколько данных. Используется в графике, ML, медиа.

**Многоядерность и SMT (hyper-threading)** — несколько ядер на одном чипе, каждое может выполнять 1–2 потока одновременно.

### Иерархия памяти

Память организована в виде пирамиды: чем выше — тем быстрее и дороже, чем ниже — тем больше объём.

| Уровень | Скорость | Размер |
|---------|----------|--------|
| Регистры | < 1 нс | < 1 КБ |
| L1 кэш | ~1 нс | ~32 КБ |
| L2 кэш | ~5 нс | ~256 КБ |
| L3 кэш | ~20 нс | ~10 МБ |
| DRAM | ~80 нс | ГБ |
| NVMe SSD | ~30 мкс | ТБ |
| SATA SSD | ~150 мкс | ТБ |
| HDD | ~10 мс | ТБ |
| Лента | секунды | ПБ |

Производительность современных приложений часто упирается не в CPU, а в **локальность доступа к памяти**. Если данные «влезают» в L1/L2 — всё летает; если каждый раз идём в DRAM — CPU простаивает.

### Устройства ввода/вывода

| Класс | Примеры |
|-------|---------|
| Ввод | Клавиатура, мышь, тачпад, микрофон, сканер, камера |
| Вывод | Монитор, принтер, динамики |
| Универсальные | Дисплей с тачскрином, VR-шлем |
| Хранение | HDD, SSD, NVMe, USB-flash, оптические диски |
| Сетевые | Ethernet, Wi-Fi, Bluetooth, 5G модемы |
| Спецустройства | GPU, TPU, DSP, FPGA, ASIC, акселераторы AI |

### Способы взаимодействия CPU и I/O

**Опрос (polling).** CPU регулярно проверяет статус устройства. Просто, но прожорливо — CPU тратит время на ожидание.

**Прерывания (IRQ).** Устройство сигнализирует CPU специальной линией. CPU прерывает текущую работу, переключается на обработчик прерывания (ISR), затем возвращается. Эффективно, но переключение контекста имеет цену.

**DMA (Direct Memory Access).** Устройство пишет в память напрямую без участия CPU. CPU получает прерывание только после завершения операции. Так работают сетевые карты, диски.

**Memory-mapped I/O.** Регистры устройства видны как обычные адреса памяти. Чтение/запись по этим адресам обращается к устройству.

**Port I/O.** Отдельное адресное пространство для устройств, специальные инструкции (`IN`, `OUT` на x86).

### Шины

Старые: ISA, PCI.

Современные:

- **PCIe** — для GPU, NVMe-дисков, акселераторов.
- **DMI / QPI / UPI / Infinity Fabric** — связь CPU и чипсета, между ядрами.
- **USB** — универсальная последовательная шина.
- **SATA** — для дисков.
- **NVMe** — поверх PCIe, для современных SSD.
- **Thunderbolt** — высокопроизводительная универсальная шина.

### Альтернативные архитектуры

**Гарвардская архитектура** — раздельные шины и память для команд и данных. Используется в DSP и большинстве микроконтроллеров (AVR, ARM Cortex-M). Преимущество — одновременный доступ к коду и данным.

**Модифицированная гарвардская архитектура** — раздельные кэши L1 для команд (I-cache) и данных (D-cache), общая RAM. Это компромисс, который используется в большинстве современных CPU (x86, ARM Cortex-A).

**CISC vs RISC.** CISC (Complex Instruction Set Computer) — много сложных инструкций (x86, x86-64). RISC (Reduced Instruction Set Computer) — мало простых, конвейеризация эффективнее (ARM, MIPS, RISC-V). Современный x86-64 внутри декодирует свои сложные инструкции в более простые «micro-ops», т.е. фактически работает как RISC.

**VLIW** — очень длинное командное слово, параллелизм задаётся компилятором (Itanium, исторически).

**Векторные процессоры** — операции над массивами. Используется в суперкомпьютерах.

### Тренды

- **Гетерогенные вычисления** — CPU + GPU + AI-ускорители (NPU, TPU) в одном чипе.
- **Chiplets** — большие процессоры собираются из небольших чиплетов (AMD Ryzen с 2019, Intel Meteor Lake).
- **ARM на серверах** — AWS Graviton, Ampere Altra конкурируют с x86.
- **ARM на десктопах** — Apple Silicon (M1/M2/M3/M4) показал, что ARM может быть быстрее и энергоэффективнее x86.
- **RISC-V** — открытая ISA, активно развивается; в РФ Байкал и Эльбрус — отечественные процессоры.
- **Persistent memory** — Intel Optane показал концепт, ушёл, но идея «постоянной памяти» с свойствами RAM продолжается.
- **CXL** — стандарт связи CPU и акселераторов с когерентным доступом к памяти.

### Что важно сказать в итоге

Архитектура фон Неймана — фундамент всей современной компьютерной техники: программа и данные в одной памяти, последовательное исполнение, двоичное кодирование. Базовые компоненты — CPU, память, I/O, шины. Современный CPU — это сложный механизм с конвейером, out-of-order execution, кэшами, MMU, многоядерностью. Память организована иерархически от регистров до диска. I/O взаимодействует с CPU через прерывания и DMA. Альтернатива — гарвардская архитектура (DSP, MCU), RISC vs CISC. Тренды 2026 — гетерогенные вычисления, chiplets, ARM на серверах и десктопах, RISC-V.
