## Развёрнутый ответ

### Что такое Linux

**Linux** — это Unix-подобная операционная система с открытым исходным кодом. Точнее, **Linux — это ядро**, написанное Линусом Торвальдсом в 1991 году. То, что обычно называют «Linux», на самом деле — **дистрибутив**: ядро + GNU userland (coreutils, bash, glibc) + пакетный менеджер + системные сервисы (systemd) + графическая среда.

Линус начал проект как студенческий эксперимент — портировать терминал MINIX на свой 386. За полтора года получилось рабочее ядро, которое он выложил под лицензией **GPLv2**. С этого начался один из самых успешных open-source-проектов в истории. Сегодня в ядре Linux работают тысячи разработчиков из сотен компаний.

### Философия Unix

Linux наследует **философию Unix**, сформулированную в 1970-х в Bell Labs:

- **Делай одну вещь, и делай её хорошо.** Каждая утилита — это маленькая программа с чёткой задачей.
- **Программы работают вместе.** Стандартный ввод-вывод и **pipes** позволяют комбинировать утилиты в цепочки.
- **Текст — универсальный интерфейс.** Конфиги, логи, обмен — в текстовом виде.

Эта философия видна в командной строке: `cat access.log | grep 404 | awk '{print $7}' | sort | uniq -c | sort -rn | head` — пять небольших утилит, склеенных в одну строку, делают то, для чего в Windows нужно отдельное приложение.

### Архитектура

Ядро Linux — **монолитное с поддержкой модулей**. То есть основные подсистемы скомпилированы в один бинарник ядра (`vmlinuz`), но **драйверы и многие фичи могут подгружаться как модули** (`.ko` файлы) в рантайме:

```bash
lsmod                    # список загруженных модулей
sudo modprobe nvidia     # загрузить модуль
sudo rmmod nvidia        # выгрузить
```

Это даёт компромисс: производительность монолита (всё в одной адресной зоне) + гибкость (можно не тащить ненужные драйверы).

Архитектурно Linux выглядит так:

```
User space:  приложения, шеллы (bash, zsh), GUI (Wayland/X11)
              │ системные вызовы / glibc
Kernel:       VFS │ драйверы │ планировщик │ MM │ сеть │ IPC
              └─ ядерные модули (.ko)
Hardware:     CPU, RAM, диски, сеть
```

### Подсистемы ядра

**Процессы и планировщик.** Управление потоками выполнения. Современный планировщик — **EEVDF** (с Linux 6.6), сменивший CFS. Поддерживает реальное время через SCHED_FIFO/SCHED_RR.

**Управление памятью.** Страничная виртуальная память, swap, slab/slub аллокаторы для ядра, transparent huge pages, **cgroups** для ограничения ресурсов процессов.

**VFS (Virtual File System).** Универсальный интерфейс к файловым системам: ext4, XFS, Btrfs, ZFS, F2FS, FAT, NTFS, NFS, SMB. Программа просто вызывает `read`/`write`, не зная, на какой ФС лежит файл.

**Сеть.** Полный стек TCP/IP, **netfilter** (база iptables/nftables), namespaces, **eBPF** для программируемой обработки пакетов.

**Безопасность.** Пользователи/группы (DAC), **capabilities** (тонкая разбивка прав root), **SELinux**, **AppArmor** (MAC), **audit framework**.

**IPC.** Пайпы, сокеты (включая Unix-сокеты), сигналы, shared memory, semaphores, message queues.

**Системы инициализации.** **systemd** — современный стандарт (`systemctl`, `journalctl`, юниты). Альтернативы: OpenRC (Gentoo, Alpine), SysV init.

### Дистрибутив = ядро + всё остальное

Различия дистрибутивов — в:

- **Пакетном менеджере**: apt/dpkg (Debian, Ubuntu), dnf/rpm (RHEL, Fedora), pacman (Arch), zypper (SUSE), apk (Alpine).
- **Userland**: GNU coreutils почти везде, в Alpine — busybox.
- **Системе инициализации**: systemd в большинстве, OpenRC и Runit в нишах.
- **Сервисах и приложениях по умолчанию**.
- **Графической среде** (DE): GNOME, KDE Plasma, XFCE, Cinnamon, Hyprland, Sway.
- **Целевой аудитории**: серверы, десктоп, embedded.

Популярные семейства:

- **Debian / Ubuntu** — десктоп и серверы, очень популярны.
- **Red Hat / Fedora / RHEL / Rocky / AlmaLinux** — enterprise-серверы.
- **Arch** — для тех, кто хочет максимум контроля.
- **SUSE / openSUSE** — европейский enterprise.
- **Alpine** — минималистичный, часто в Docker-контейнерах.

В России:

- **Astra Linux** — с акцентом на безопасность (есть сертификация по гостайне).
- **RED OS** — отечественный enterprise-дистрибутив.
- **ALT Linux** — на базе Sisyphus, для desktop и серверов.

### Файловая структура (FHS — Filesystem Hierarchy Standard)

```
/            корень
/bin /sbin   бинарники (на современных системах часто симлинки в /usr)
/etc         конфигурационные файлы
/home        домашние каталоги пользователей
/var         переменные данные (логи, БД, кеш)
/usr         программы и библиотеки
/tmp         временные файлы
/proc        виртуальная ФС с информацией о процессах
/sys         виртуальная ФС с информацией о ядре
/dev         устройства
/mnt /media  точки монтирования
/opt         сторонние приложения
```

**Всё — это файл** — ещё один принцип Unix. Устройства, процессы, сетевые сокеты — всё представлено через файловую систему.

### Контейнеры и виртуализация

Linux лежит в основе современной контейнерной инфраструктуры:

- **cgroups** — ограничение ресурсов;
- **namespaces** — изоляция (PID, NET, MNT, USER, UTS, IPC);
- эти два механизма — основа Docker, Podman, Kubernetes;
- **KVM** — гипервизор уровня ядра, базовый для QEMU, Proxmox, libvirt, OpenStack;
- **eBPF** — программируемая телеметрия и сеть, используется в Cilium, Falco, observability-инструментах.

### Базовые команды

```bash
ls -lah                # листинг с правами и размерами
ps -ef                 # процессы
top / htop / btop      # мониторинг в реальном времени
df -h ; du -sh *       # диск
free -h                # память
ip a ; ip route        # сетевые интерфейсы и маршруты
systemctl status nginx # юнит systemd
journalctl -u nginx    # логи службы
grep -R "pattern" .    # поиск по содержимому
chmod 644 file ; chown user:group file
ssh user@host          # удалённое подключение
```

### Где применяется Linux

- **Серверы** — фактический стандарт. Около 80% веба работает на Linux.
- **Облака** — все крупные провайдеры (AWS, Azure, GCP, Yandex Cloud) используют Linux.
- **Android** — построен на ядре Linux.
- **Embedded** — Yocto, Buildroot собирают Linux под конкретное железо.
- **Суперкомпьютеры** — 100% Top500 работают на Linux.
- **Сетевое оборудование** — большинство роутеров и коммутаторов.
- **Desktop** — небольшой, но растущий сегмент; в России активно растёт за счёт импортозамещения.

### Сильные и слабые стороны

**Сильные стороны:** открытый код, отсутствие лицензионных платежей, гибкость, стабильность, безопасность, огромное сообщество, поддержка любого железа.

**Слабые стороны:** фрагментация дистрибутивов и пакетных форматов, меньшая поддержка специализированного коммерческого ПО (Adobe, ряд игр), порог входа выше Windows/macOS.

### Что важно сказать в итоге

Linux — это Unix-подобное ядро с открытым исходным кодом, разработанное Линусом Торвальдсом и поддерживаемое огромным сообществом. Архитектурно — монолитное ядро с модулями, наследует философию Unix («делай одно дело хорошо», «всё — файл», pipes). На основе ядра создаются дистрибутивы — ядро + GNU userland + пакетный менеджер + DE. Linux доминирует на серверах, в облаках, на мобильных устройствах (Android), в embedded и суперкомпьютерах. Сильные стороны — открытость, гибкость, стабильность; слабые — фрагментация и порог входа.
