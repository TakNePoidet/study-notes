## Развёрнутый ответ

### Что такое файловая система

**Файловая система (ФС)** — это способ организации, хранения и доступа к данным на носителе. ФС определяет:

- какие структуры данных используются для хранения метаданных;
- как располагаются файлы на диске;
- как осуществляется поиск, чтение, запись;
- какие операции поддерживаются (атрибуты, права, ссылки);
- как обеспечивается надёжность при сбоях.

Без ФС диск — это просто непрерывный массив байт. ФС превращает его в иерархическую структуру файлов и каталогов с правами и метаданными.

### Структура файла в Unix

В Unix-системах файл устроен через несколько слоёв.

**Inode** — структура с метаданными файла: размер, времена создания/доступа/изменения, uid/gid владельцев, права, тип файла, указатели на блоки данных. У inode есть номер; через номер ОС находит файл.

**Каталог** — это специальный файл, содержащий **таблицу** пар «имя → номер inode». Когда мы говорим «открыть файл по имени», ОС находит каталог, читает таблицу, получает номер inode, по нему — метаданные и данные.

**Блоки данных** — собственно содержимое файла, расположенное в блоках фиксированного размера (обычно 4 КБ).

Из этой архитектуры вытекают важные следствия:

**Жёсткая ссылка** — это **ещё одна запись в каталоге**, указывающая на тот же inode. Файл считается удалённым, только когда число ссылок становится нулём. Жёсткие ссылки работают только внутри одной ФС.

**Символическая ссылка** — это отдельный файл со специальным типом, содержащий путь к другому файлу. Может вести «куда угодно», в том числе на другие ФС или несуществующие пути.

### Популярные файловые системы

| ФС | Особенности |
|----|-------------|
| **ext4** | Стандарт Linux, журналируемая, до 1 EiB |
| **XFS** | Высокая производительность для больших файлов, журнал, online resize |
| **Btrfs** | Copy-on-write, снимки, subvolumes, сжатие, чек-суммы |
| **ZFS** | CoW, RAID-Z, дедупликация, чек-суммы, snapshots, send/receive |
| **F2FS** | Оптимизирована под flash-память (SSD, eMMC) |
| **NTFS** | Стандарт Windows, права через ACL, журналируемая |
| **APFS** | macOS, CoW, snapshots, шифрование |
| **FAT32 / exFAT** | Простые, для флешек, без прав |
| **tmpfs** | В оперативной памяти, очень быстрая |
| **NFS / SMB** | Сетевые ФС |
| **Ceph, GlusterFS, Lustre** | Распределённые/кластерные |

### Защита файлов

**Стандартные права Unix.** Для каждого файла определены три набора прав: для **владельца (user)**, **группы**, **всех остальных (other)**. В каждом наборе три бита — `r` (чтение), `w` (запись), `x` (исполнение/вход в каталог).

```bash
ls -l
# -rw-r--r--  1 matvey users  4096 May 15 10:00 notes.md
#  ↑ тип  ↑user ↑group ↑other
```

Запись прав в **восьмеричной нотации**: каждый набор — 3 бита, итого 9 бит → 3 восьмеричные цифры. `755` = `rwxr-xr-x`, `644` = `rw-r--r--`, `600` = `rw-------`.

```bash
chmod 640 notes.md           # rw- для владельца, r-- для группы, нет для остальных
chown matvey:users notes.md  # сменить владельца и группу
chmod +x script.sh           # +x для всех
chmod g+w notes.md           # +w только для группы
```

Дополнительные биты: **SUID/SGID** (запускать с правами владельца/группы файла), **sticky bit** (в `/tmp` — нельзя удалить чужие файлы).

**ACL (Access Control Lists).** Стандартные права Unix не позволяют дать доступ конкретным пользователям/группам помимо трёх категорий. ACL добавляют это:

```bash
setfacl -m u:alice:rw notes.md
getfacl notes.md
```

**Расширенные атрибуты (xattr).** Произвольные key=value метаданные. Используется для меток SELinux, защиты от изменения (`chattr +i`), и прикладных нужд.

```bash
chattr +i critical.conf   # immutable — нельзя удалить даже root
lsattr critical.conf
```

**MAC (Mandatory Access Control).** Поверх DAC (дискреционных прав) добавляются обязательные политики, которые пользователь не может изменить:

- **SELinux** — метки на каждый объект, политики на уровне ОС. Активен по умолчанию в RHEL, Fedora.
- **AppArmor** — путь-ориентированные профили. Используется в Ubuntu.

MAC защищает даже от случаев, когда злоумышленник получил root: процесс ограничен своей политикой.

### Шифрование файлов

**LUKS / dm-crypt** — шифрование на уровне блочного устройства. Весь раздел зашифрован, прозрачно для пользователя.

**fscrypt** — шифрование на уровне ФС (ext4, F2FS). Каждый каталог может иметь свой ключ.

**eCryptfs / EncFS / gocryptfs** — пользовательские слои поверх ФС.

**BitLocker** (Windows), **FileVault** (macOS) — встроенные решения для дисковой шифровки.

### Управление файлами на практике

```bash
touch file.txt              # создать пустой файл
mkdir -p dir/sub            # создать каталоги
cp -a src dst               # копировать с сохранением атрибутов
mv old new                  # переместить или переименовать
rm -rf path                 # удалить рекурсивно
ln file hard.txt            # жёсткая ссылка
ln -s file sym.txt          # символическая ссылка
find /home -type f -mtime -1 -name '*.log'   # найти файлы изменённые за сутки
stat file.txt               # метаданные
mount /dev/sdb1 /mnt/usb    # монтирование
umount /mnt/usb             # размонтирование
```

### Журналирование и целостность

**Журналируемые ФС** (ext4, NTFS, XFS) ведут журнал намерений: «собираюсь обновить такие-то блоки». При сбое после перезагрузки журнал воспроизводится — это защищает от полу-записанных операций.

**Checksumming** в Btrfs и ZFS — для каждого блока хранится контрольная сумма. При чтении сверяется; если не совпала — bit rot обнаружен, и при наличии RAID-Z данные восстанавливаются.

**RAID** — избыточность на уровне дисков:
- RAID-0 — striping без избыточности (только скорость);
- RAID-1 — зеркало;
- RAID-5/6 — с чётностью;
- RAID-10 — комбинация зеркал и striping.

**Snapshots** — мгновенные срезы (Btrfs, ZFS, LVM). Удобно для бэкапов и тестов.

### Квоты

`quota`, `xfs_quota` — ограничения на пользователя/группу по объёму и числу inode. Полезно на shared-системах и в учебных средах.

### Прочие особенности

**VFS (Virtual File System)** в Linux абстрагирует разные ФС за единым API: программе всё равно, на какой ФС лежит файл.

**/proc и /sys** — виртуальные ФС. `/proc/cpuinfo` — данные о CPU, `/proc/[PID]/status` — состояние процесса, `/sys` — древо устройств и параметров ядра.

**FUSE (Filesystem in Userspace)** — позволяет реализовать ФС в user space без правки ядра. Так работают sshfs (доступ к удалённой ФС через SSH), gocryptfs, NTFS-3G и многие другие.

### Безопасность данных

Чтобы данные были защищены, нужно:

- регулярные **бэкапы** (правило 3-2-1: 3 копии, 2 разных носителя, 1 — оффсайт);
- **шифрование чувствительных данных** в покое;
- **антивирус** для шар, к которым обращаются Windows-клиенты;
- **очистка перед утилизацией** дисков (`shred`, secure-erase для SSD);
- **журналирование событий доступа** для аудита.

### Что важно сказать в итоге

Файловая система организует хранение данных на носителе через каталоги, inode и блоки данных. Современный Linux поддерживает множество ФС: ext4 как стандарт, XFS для производительности, Btrfs и ZFS для продвинутых возможностей (CoW, снимки, чек-суммы), tmpfs для оперативной памяти, NFS и SMB для сети. Защита файлов строится на нескольких уровнях: стандартные права Unix, ACL, расширенные атрибуты, обязательный контроль (SELinux/AppArmor), шифрование. Управление файлами — это базовые команды и понимание структуры (inode, ссылки, монтирование). Современные ФС добавляют журналирование, контрольные суммы, snapshots и RAID для надёжности.
