## Развёрнутый ответ

### Общая характеристика SQL

**SQL (Structured Query Language)** — это стандартизованный декларативный язык запросов к реляционным базам данных. Декларативность здесь принципиальная особенность: я описываю **что** хочу получить, а не **как** именно нужно бежать по таблицам. Оптимизатор СУБД сам строит план выполнения, используя статистику, индексы и эвристики.

SQL появился в 1970-х годах в IBM (изначально назывался SEQUEL), и в 1986 году был принят первый стандарт ANSI/ISO. С тех пор стандарт развивается, добавляя возможности — от подзапросов и CTE до окон, JSON и темпоральных таблиц. Сейчас актуальные стандарты — SQL:2016 и SQL:2023.

SQL — обязательное знание для любого, кто работает с данными: разработчика, аналитика, инженера ML.

### Четыре подъязыка SQL

Сам по себе SQL делится на четыре больших раздела по назначению.

**DDL (Data Definition Language)** — язык определения данных. Сюда входят команды, описывающие схему: `CREATE`, `ALTER`, `DROP`, `TRUNCATE`. Это операторы, через которые мы создаём таблицы, добавляем столбцы, строим индексы, удаляем объекты. DDL-команды обычно автоматически фиксируются (commit) после выполнения.

**DML (Data Manipulation Language)** — язык работы с данными: `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `MERGE`. Это то, чем пользователи и приложения занимаются 99% времени. Сюда же традиционно относят и `SELECT`, хотя в некоторых классификациях его выделяют отдельно как DQL — Data Query Language.

**DCL (Data Control Language)** — управление доступом: `GRANT` и `REVOKE`. Эти команды раздают и отзывают права на объекты — кто может читать таблицу, кто записывать, кто выполнять процедуры.

**TCL (Transaction Control Language)** — управление транзакциями: `BEGIN` (или `START TRANSACTION`), `COMMIT`, `ROLLBACK`, `SAVEPOINT`. Транзакции обеспечивают ACID-свойства: атомарность изменений, согласованность, изоляцию параллельных операций и долговечность зафиксированных данных.

### Базовый синтаксис SELECT

Главный оператор DML — `SELECT`. Его базовая структура такая:

```sql
SELECT поля
FROM источник
[JOIN другая_таблица ON условие]
[WHERE условие_фильтрации]
[GROUP BY поля_группировки]
[HAVING условие_по_группам]
[ORDER BY сортировка]
[LIMIT n OFFSET m];
```

Важно понимать, что **порядок написания и порядок логического выполнения отличаются**. Пишем мы в порядке `SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT`, а выполняется логически:

1. `FROM` и `JOIN` — определяется множество строк-источников;
2. `WHERE` — фильтрация по строкам;
3. `GROUP BY` — группировка;
4. `HAVING` — фильтрация уже по группам;
5. `SELECT` — вычисление выражений и формирование выходного набора;
6. `ORDER BY` — сортировка результата;
7. `LIMIT` / `OFFSET` — отсечение страницы.

Это понимание критично: например, в `WHERE` нельзя использовать алиасы из `SELECT`, потому что `WHERE` выполняется раньше `SELECT`.

### Типы запросов

**Скалярные запросы.** Возвращают одно значение: `SELECT NOW()`, `SELECT 1 + 1`.

**Однотабличные.** Простая выборка из одной таблицы с фильтрацией.

**С соединениями (JOIN).** Объединяют несколько таблиц. Виды: `INNER JOIN` (только совпавшие строки), `LEFT/RIGHT OUTER JOIN` (включая несовпавшие из левой/правой таблицы), `FULL OUTER JOIN` (всё), `CROSS JOIN` (декартово произведение).

**С подзапросами.** Подзапросы могут возвращать скаляр, строку, набор. Различают **скоррелированные** (зависят от внешнего запроса, выполняются на каждую строку) и **нескоррелированные** (выполняются один раз). Часто заменяются на JOIN или CTE.

**Агрегатные.** Используют `GROUP BY` и агрегатные функции (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`, `STRING_AGG`, `ARRAY_AGG`).

**Оконные функции.** Появились в SQL:2003, позволяют считать агрегаты «по окну» без сворачивания строк — `ROW_NUMBER()`, `RANK()`, `LAG()`, `LEAD()`, скользящие суммы. Очень мощный инструмент аналитики.

**Рекурсивные CTE (`WITH RECURSIVE`).** Позволяют обходить иерархии и графы внутри одного запроса.

### Пример со всеми основными конструкциями

```sql
-- DDL: создаём схему
CREATE TABLE orders (
  id          SERIAL PRIMARY KEY,
  user_id     INT NOT NULL REFERENCES users(id),
  total       NUMERIC(10,2) NOT NULL CHECK (total >= 0),
  created_at  TIMESTAMP DEFAULT NOW()
);

-- DML: топ-5 пользователей по выручке в 2026 году
SELECT u.name, SUM(o.total) AS revenue
FROM orders o
JOIN users u ON u.id = o.user_id
WHERE o.created_at >= '2026-01-01'
GROUP BY u.name
HAVING SUM(o.total) > 0
ORDER BY revenue DESC
LIMIT 5;

-- Оконная функция: ранг каждого заказа пользователя по времени
SELECT id, user_id, created_at,
       ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY created_at) AS n
FROM orders;

-- Рекурсивный CTE: иерархия категорий
WITH RECURSIVE tree AS (
  SELECT id, parent_id, name, 1 AS lvl
    FROM categories WHERE parent_id IS NULL
  UNION ALL
  SELECT c.id, c.parent_id, c.name, t.lvl + 1
    FROM categories c JOIN tree t ON c.parent_id = t.id
)
SELECT * FROM tree;

-- TCL: атомарный перевод денег
BEGIN;
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
```

### Версии стандарта SQL

Стандарт развивался волнами, и каждая версия привнесла что-то важное:

- **SQL-86/89** — первый официальный стандарт, базовый синтаксис.
- **SQL-92** — стал «канонической базой», большинство учебников опираются на него.
- **SQL:1999** — добавил рекурсивные CTE, триггеры, объектные расширения.
- **SQL:2003** — оконные функции, поддержка XML, оператор `MERGE`.
- **SQL:2011** — темпоральные таблицы (хранение истории изменений).
- **SQL:2016** — поддержка JSON, polymorphic table functions.
- **SQL:2023** — графовые запросы (SQL/PGQ), улучшения JSON.

### Диалекты SQL

Стандарт описывает только базу; каждая СУБД добавляет свои расширения, называемые **диалектами**:

- **PostgreSQL** — массивы, JSONB, оконные функции «из коробки», CTE, расширяемые типы.
- **Oracle PL/SQL** — мощный процедурный язык, иерархия через `CONNECT BY`, аналитика.
- **MS SQL T-SQL** — `TOP`, специфичный синтаксис, мощные оконные функции.
- **MySQL** — упрощённый SQL, исторически слабый по транзакциям на MyISAM, постепенно догоняет.
- **SQLite** — минималистичный, встраиваемый.
- **ClickHouse** — диалект для OLAP с массивами, лямбдами и специфическими функциями.

При переходе между диалектами нужно учитывать различия в синтаксисе (`LIMIT` vs `TOP`, `RETURNING` vs `OUTPUT`), доступных типах данных, поведении NULL.

### Что важно сказать в итоге

SQL — это универсальный декларативный язык работы с реляционными данными, разделённый на четыре подъязыка (DDL, DML, DCL, TCL). Его сила — в простоте основ и мощи продвинутых конструкций: JOIN, подзапросы, CTE, оконные функции. Версии стандарта развиваются десятилетиями, при этом каждая СУБД реализует свой диалект с дополнительными возможностями. Уверенное владение SQL — базовый навык для любого специалиста, работающего с данными.
