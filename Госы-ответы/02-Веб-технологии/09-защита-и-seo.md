## Развёрнутый ответ

### Угрозы и защита при работе в интернете

Когда пользователь и веб-приложение обмениваются данными по сети, на каждом шаге существуют угрозы. Перехват трафика, MITM-атаки (man-in-the-middle), фишинговые сайты, XSS, CSRF, SQL-инъекции, brute-force и подбор паролей — это далеко не полный список.

Защита строится по нескольким уровням, что называется **defense in depth** — эшелонированная защита.

**Транспортный уровень.** Использование HTTPS (TLS 1.2/1.3) шифрует трафик и предотвращает MITM. Дополнительные меры: HSTS-заголовок (`Strict-Transport-Security`), который заставляет браузер всегда использовать HTTPS; обновление сертификатов; принцип «TLS only».

**Аутентификация и пароли.** Длинные парольные фразы, менеджеры паролей, обязательная **многофакторная аутентификация (MFA)** — TOTP, push-уведомления, аппаратные ключи (YubiKey, FIDO2/WebAuthn). На стороне сервера пароли хранятся только в виде хешей с солью — bcrypt, argon2, scrypt.

**Защита веб-приложения.** Здесь работает классический список OWASP Top 10:

- **SQL-инъекции** — защита через параметризованные запросы, ORM, недопущение конкатенации строк с пользовательским вводом;
- **XSS (Cross-Site Scripting)** — экранирование вывода, заголовок **Content Security Policy (CSP)**, который ограничивает источники скриптов;
- **CSRF (Cross-Site Request Forgery)** — токены в формах, флаги cookie `SameSite=Lax/Strict`;
- **SSRF (Server-Side Request Forgery)** — валидация URL, белые списки;
- **IDOR (Insecure Direct Object Reference)** — серверная авторизация на каждый запрос, нельзя доверять клиенту;
- **Broken Access Control** — проверять права на сервере, не скрывать функционал только UI-логикой.

**Сетевая защита.** Файрволы, **WAF (Web Application Firewall)** — Cloudflare, ModSecurity, проверяющие запросы по сигнатурам атак, ограничение частоты запросов (rate limiting), системы обнаружения и предотвращения вторжений (IDS/IPS).

**Организационные меры.** Принцип наименьших привилегий (least privilege), регулярные бэкапы, мониторинг и SIEM, план реагирования на инциденты, обучение сотрудников против фишинга.

### Как работают поисковые системы

Поисковая система — это сложный распределённый комплекс, который условно делится на четыре этапа.

**1. Крaулинг (Crawling).** Робот поисковой системы (Googlebot, YandexBot, Bingbot) обходит ссылки, скачивает страницы и помещает их в очередь обработки. Робот учитывает указания файла `robots.txt` (что можно и нельзя обходить), карту сайта `sitemap.xml` и мета-теги `noindex`/`nofollow`.

**2. Индексация (Indexing).** Скачанные страницы обрабатываются: извлекается текст, удаляются HTML-теги, нормализуются слова (лемматизация, стемминг), убираются стоп-слова. Результат заносится в **инвертированный индекс** — структуру, где каждому термину сопоставлен список документов, в которых он встречается.

**3. Ранжирование (Ranking).** Когда пользователь вводит запрос, поисковик подбирает релевантные документы из индекса и сортирует их по сотням факторов: текстовая релевантность (BM25, TF-IDF), авторитет сайта (PageRank и его наследники), поведенческие сигналы (CTR, время на странице, возвраты к выдаче), свежесть, скорость загрузки, адаптивность под мобильные. Современные поисковики используют ML-модели вроде BERT, MUM, нейросетевые модели Yandex Y1/Y2 для понимания смысла запроса.

**4. Выдача (Serving).** Пользователь видит SERP — страницу результатов с органическими ссылками, рекламой, расширенными сниппетами (rich snippets), быстрыми ответами, картинками и видео.

### Что такое SEO

**SEO (Search Engine Optimization)** — это комплекс мер по улучшению видимости сайта в органической выдаче поисковых систем. SEO даёт долгосрочный канал трафика, в отличие от платной рекламы, но требует времени (от 3–6 месяцев до года) и постоянной работы.

SEO делится на три большие группы.

### On-page SEO — оптимизация на самой странице

- **Контент.** Уникальный, отвечающий на интент пользователя. Поисковые системы умеют отличать переписанный контент от ценного. Хорошая статья закрывает вопрос пользователя полно, а не размазывает воду.
- **Заголовки и мета-теги.** `<title>` — главный сигнал релевантности, до 60 символов. `<meta name="description">` — описание для сниппета, до 160 символов. Заголовки H1–H3 структурируют текст.
- **Семантическая разметка.** Использование structured data (Schema.org, JSON-LD) для специальных сниппетов: рейтинги, рецепты, мероприятия, FAQ, цены товаров.
- **Внутренняя перелинковка.** Ссылки между страницами сайта передают авторитет и помогают поисковику понять структуру.
- **Понятные URL.** `example.com/blog/seo-guide` лучше, чем `example.com/?p=1234`.
- **Скорость загрузки.** Современная метрика — **Core Web Vitals**: LCP (Largest Contentful Paint, ≤ 2.5 сек), INP (Interaction to Next Paint, заменил FID), CLS (Cumulative Layout Shift, ≤ 0.1).
- **Адаптивность.** Мобильная версия не должна уступать десктопу; Google применяет mobile-first indexing.

### Технический SEO

- `robots.txt` — какие разделы можно индексировать.
- `sitemap.xml` — карта сайта для поисковика.
- `canonical` — указатель основной версии страницы при дубликатах.
- `hreflang` — версии страницы для разных языков и регионов.
- Корректные 301-редиректы вместо 404.
- HTTPS обязательно.
- Отсутствие дублей, корректная пагинация.
- Здоровая структура URL и логи crawl-бюджета.

### Off-page SEO — то, что вне сайта

- **Backlinks** — внешние ссылки с других сайтов. Качество важнее количества: одна ссылка с авторитетного сайта весит больше сотни с спам-площадок.
- **Упоминания бренда** даже без ссылок учитываются как сигналы доверия.
- **Локальное SEO** — регистрация в Google Business Profile / Яндекс.Бизнес, отзывы, корректные NAP-данные (Name, Address, Phone).
- **Поведенческие факторы** на выдаче — высокий CTR сниппета, низкий показатель отказов, возвраты к выдаче.

### Пример JSON-LD разметки статьи

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Что такое SEO",
  "author": { "@type": "Person", "name": "Матвей" },
  "datePublished": "2026-05-15"
}
</script>
```

### Чёрное SEO и риски

Существуют и **чёрные методы**, которыми пытаются обмануть алгоритмы: клоакинг (разный контент роботу и пользователю), массовая покупка ссылок, ссылочные фермы, дорвеи, скрытый текст того же цвета, что фон. Современные алгоритмы (Google Penguin, Яндекс «Минусинск», «Баден-Баден», «Прокси») быстро распознают такие практики и накладывают санкции вплоть до полного исключения сайта из выдачи.

### Что важно сказать в итоге

Защита веб-приложения — это многоуровневая задача: транспортное шифрование (HTTPS), безопасная аутентификация (MFA), защита от OWASP Top 10, сетевые фильтры (WAF) и организационные меры. Поисковые системы работают по схеме crawling → indexing → ranking → serving, и SEO — это набор мер, помогающих сайту хорошо ранжироваться. Делится на on-page (контент, теги, скорость), off-page (ссылки, упоминания, локальное) и технический (robots, sitemap, canonical, Core Web Vitals). Хорошее SEO — это инвестиция в долгосрочный органический трафик, чёрные методы дают быстрый эффект, но приводят к санкциям.
