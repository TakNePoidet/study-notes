## Развёрнутый ответ

### Типы данных в JavaScript

В JavaScript принято делить значения на два больших класса: **примитивы** и **объекты**. Примитивы хранятся **по значению**, объекты — **по ссылке**, и это важно понимать при сравнении и передаче в функции.

Примитивы:

- `string` — строки;
- `number` — числа (включая целые и с плавающей точкой; в спецификации только одно числовое представление — IEEE 754 double);
- `bigint` — большие целые, литерал с суффиксом `n`;
- `boolean` — `true` / `false`;
- `undefined` — переменная объявлена, но не инициализирована;
- `null` — отсутствие значения, явный «пустой» результат;
- `symbol` — уникальный идентификатор, появился в ES2015.

Объекты — всё остальное: `Object`, массивы (`Array`), функции (`Function`), даты (`Date`), регулярные выражения, типизированные массивы, специальные коллекции `Map`, `Set`, `WeakMap`, `WeakSet`.

Маленький курьёз: `typeof null === 'object'` — это историческая ошибка из самой первой версии языка, которую сохранили из соображений совместимости.

### Переменные

В JS три способа объявления переменных, и между ними есть существенные различия.

- **var** — старый способ, имеет **функциональную область видимости**, поднимается (hoisting) в начало области, и можно переприсваивать и повторно объявлять. Сегодня практически не используется.
- **let** — современный способ для изменяемых переменных. Имеет **блочную область** (видна только внутри `{}`, в которых объявлена), не поднимается так, чтобы можно было использовать до объявления (зона **temporal dead zone**).
- **const** — для констант. Блочная область, нельзя переприсваивать. Важно: `const` не делает значение immutable, если это объект — содержимое можно менять, нельзя только переприсвоить саму переменную.

Современное правило: **по умолчанию `const`, при необходимости `let`, `var` не использовать**.

### Структуры данных

**Объект** — это набор пар ключ → значение. Ключи — строки или символы. Объекты — основная структура для представления сущностей в JS.

```js
const user = {
  name: 'Matvey',
  age: 22,
  greet() { return `Hi, ${this.name}`; }
};
console.log(user.name, user.greet());
```

**Массив** — упорядоченная коллекция, индексированная числами. Под капотом это всё тот же объект, но с особым поведением. У массивов есть мощный набор методов: `map`, `filter`, `reduce`, `find`, `some`, `every`, `forEach`, `flat`, `flatMap`. Эти методы — основа функционального стиля в JS.

```js
const nums = [1, 2, 3, 4];
const doubled = nums.map(n => n * 2);          // [2, 4, 6, 8]
const sum     = nums.reduce((a, b) => a + b, 0); // 10
const evens   = nums.filter(n => n % 2 === 0);   // [2, 4]
```

**Map** — современная альтернатива объекту, когда ключи могут быть любыми (включая объекты), сохраняется порядок вставки, и есть удобные методы `set`/`get`/`has`/`delete`.

```js
const cache = new Map();
cache.set('key', 'value');
cache.get('key');
```

**Set** — коллекция уникальных значений, удобна для дедупликации и проверки наличия.

```js
const tags = new Set(['js', 'web', 'js']);  // js, web — без дублей
```

**WeakMap** и **WeakSet** — варианты, где ключи могут быть собраны сборщиком мусора. Используются для приватных данных и кеширования по объектам.

### Операторы

Стандартные арифметические (`+`, `-`, `*`, `/`, `%`, `**`), логические (`&&`, `||`, `!`), сравнения (`==` vs `===`).

Главное правило сравнения: **`==` приводит типы**, поэтому `1 == '1'` — `true`, а `0 == ''` — тоже `true`. Это источник багов. **`===` строгое сравнение**, без приведения, и его рекомендуется использовать всегда.

Современные операторы:

```js
// Логическое ИЛИ vs nullish coalescing
const a = '' || 'default';   // 'default' — '' falsy
const b = '' ?? 'default';   // ''        — ?? проверяет только null/undefined

// Optional chaining
const city = user?.address?.city;  // undefined, если что-то по пути null

// Spread / rest
const merged = { ...a, ...b };
const all    = [...arr1, ...arr2];
function sum(...nums) { return nums.reduce((a, b) => a + b, 0); }

// Деструктуризация
const { name, age = 18 } = user;
const [first, ...rest]   = arr;
```

### Функции и замыкания

Функции в JS — **объекты первого класса**: их можно сохранять в переменных, передавать как аргументы, возвращать из других функций.

```js
function add(a, b) { return a + b; }  // объявление
const sub = function(a, b) { return a - b; };  // выражение
const mul = (a, b) => a * b;          // стрелочная
```

Стрелочные функции отличаются: у них **нет своего `this`** (он берётся из окружающего контекста), нет `arguments`, нельзя вызвать с `new`. Это удобно в коллбэках, где раньше приходилось писать `function() { ... }.bind(this)`.

**Замыкания** — одна из ключевых особенностей JS. Функция запоминает область видимости, в которой была создана.

```js
function makeCounter() {
  let count = 0;
  return () => ++count;
}
const c = makeCounter();
c(); c(); c();  // 1, 2, 3
```

`count` остаётся «живым» внутри возвращённой функции, хотя `makeCounter` уже отработала.

### Классы и прототипы

Прототипное наследование — историческая база JS, на которой надстроен классовый синтаксис из ES2015:

```js
class User {
  constructor(name) { this.name = name; }
  greet() { return `Hi, ${this.name}`; }
}

class Admin extends User {
  constructor(name) { super(name); this.role = 'admin'; }
}

const u = new Admin('Matvey');
console.log(u.greet());  // Hi, Matvey
```

Под капотом классы — это всё те же функции-конструкторы и прототипы, но с приятным синтаксисом.

### Технология Ajax

**Ajax (Asynchronous JavaScript and XML)** — это не отдельная технология, а **подход**, при котором страница обменивается данными с сервером без полной перезагрузки. Идея появилась в начале 2000-х (название дал Jesse James Garrett в 2005-м) и совершила революцию: GMail, Google Maps работали именно на Ajax, и веб стал «приложениями», а не просто документами.

Сегодня формат данных, как правило, — JSON, а не XML, но название осталось.

Старая реализация — `XMLHttpRequest`:

```js
const xhr = new XMLHttpRequest();
xhr.open('GET', '/api/users/42');
xhr.onload = () => console.log(xhr.responseText);
xhr.send();
```

Современная — `fetch` + `async/await`:

```js
async function getUser(id) {
  try {
    const res = await fetch(`/api/users/${id}`, {
      headers: { 'Accept': 'application/json' }
    });
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return await res.json();
  } catch (e) {
    console.error('Ошибка запроса', e);
    return null;
  }
}
```

`fetch` возвращает **Promise**, который разрешается ответом сервера. `async/await` (ES2017) делает асинхронный код визуально похожим на синхронный.

Альтернативы и расширения:

- **axios** — популярная библиотека с интерсепторами и удобной обработкой ошибок;
- **WebSocket** — двусторонний поток данных;
- **Server-Sent Events (SSE)** — поток событий с сервера;
- **GraphQL** — декларативные запросы, фреймворки Apollo, Relay, urql;
- **AbortController** — стандартный способ отменить запрос.

### CORS

С Ajax связано важное понятие — **CORS (Cross-Origin Resource Sharing)**. Браузер по умолчанию запрещает скриптам ходить на другой домен, и сервер должен явно разрешить это заголовком `Access-Control-Allow-Origin`. Это защищает от ряда атак, но требует понимания при настройке API.

### Что важно сказать в итоге

JavaScript оперирует двумя классами значений — примитивами и объектами; современная переменная — `let` или `const`. Структуры данных: объект, массив, `Map`, `Set`. Операторы дополнены современными удобными конструкциями: spread, rest, optional chaining, nullish coalescing. Функции — объекты первого класса, замыкания — ключевая возможность. Ajax — это подход к асинхронному обмену данными с сервером без перезагрузки страницы; современный способ — `fetch` + `async/await`, дополненный CORS, отменой запросов и более продвинутыми инструментами вроде WebSocket и GraphQL.
