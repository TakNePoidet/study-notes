# 76. Информационные системы в управлении проектами

[← К группе «Корпоративные ИС»](README.md) · [← Ко всем группам](../README.md)

## План ответа

1. Что такое PMIS и зачем нужна.
2. Функциональные модули PMIS.
3. Классические системы для расписаний.
4. Agile-инструменты для гибких команд.
5. Специализированные и enterprise PPM-системы.
6. Выбор инструмента под сценарий.
7. Интеграции и тренды.

## Развёрнутый ответ

### Что такое PMIS

**PMIS (Project Management Information System)** — это программная система для планирования, исполнения, контроля и анализа проектов. PMIS объединяет данные проекта в одном месте и помогает менеджеру и команде ориентироваться в работе.

Без PMIS даже небольшой проект превращается в хаос: где задачи? кто что делает? когда дедлайн? Excel-файлы расползаются, версии теряются, статусы устаревают. PMIS решает эти проблемы централизованно.

PMIS — не одна система; это семейство решений под разные нужды:

- **классические PM-системы** для расписаний и Гантт;
- **Agile-инструменты** для команд со Scrum/Kanban;
- **корпоративные PPM** для портфельного управления;
- **специализированные** под отрасли (стройка, инженерия, ИТ).

### Функциональные модули

Современная PMIS обычно включает несколько групп функций.

**Управление проектами:**

- WBS (Work Breakdown Structure) — иерархия задач.
- Диаграмма Гантта — визуальное расписание.
- Календарное планирование, критический путь.
- Зависимости задач (FS, SS, FF, SF, лаги).

**Управление ресурсами:**

- Назначение исполнителей и оборудования.
- Загрузка и балансировка.
- Календари рабочих и выходных.

**Управление бюджетом:**

- Стоимость работ, ставки сотрудников.
- Контроль EVM: PV, EV, AC, SV, CV, SPI, CPI.
- Прогнозирование EAC (Estimate at Completion).

**Управление рисками:**

- Реестр рисков с описанием, владельцем, статусом.
- Матрица «вероятность × влияние».
- Привязка рисков к задачам.

**Управление документами:**

- Хранилище файлов с версионированием.
- Согласования, утверждения, workflow.

**Коммуникации и сотрудничество:**

- Чаты, комментарии, @-упоминания.
- Уведомления, дайджесты.
- Интеграция с почтой, мессенджерами.

**Отчётность и BI:**

- Дашборды KPI.
- Прогнозы (forecast).
- Сравнение базового и фактического плана.

**Управление портфелем (PPM, Project Portfolio Management):**

- Балансировка нескольких проектов.
- Приоритизация по стратегическим целям.
- Загрузка ресурсов на портфеле.

### Классические PMIS

**Microsoft Project / Project for the Web.** Стандарт де-факто для расписаний и Гантт-диаграмм. Знаком всем менеджерам, интегрируется с Microsoft 365, Power BI. Дёшево и привычно, особенно для классических проектов.

**Oracle Primavera P6.** Enterprise PM для строительства, нефтегаза, инфраструктуры. Тянет планы из десятков тысяч задач, поддерживает сложные ресурсные модели. Дорого и сложно, но мощно.

**OpenProj / GanttProject** — открытые альтернативы для небольших задач.

### Agile-инструменты

**Atlassian Jira** — стандарт для IT-команд. Поддерживает Scrum, Kanban, Mixed. Эпики, версии, релизы. Огромная экосистема плагинов (Tempo, Structure, eazyBI). После 2022 многим российским пользователям ограничен доступ к Jira Cloud — переходят на YouTrack или Yandex Tracker.

**Trello.** Простые канбан-доски, для небольших команд. Лёгко начать, бесплатно для базы.

**Asana, Monday.com, ClickUp, Wrike.** Универсальные task-менеджеры с разными акцентами. Asana — для команд; Monday — для визуализации; ClickUp — «всё в одном»; Wrike — для маркетинга.

**Linear.** Современный SaaS, ориентированный на продуктовые и инженерные команды. Быстрый, минималистичный UX, активно развивается.

**Notion.** Гибридный workspace: документы + базы данных + канбан + календарь. Очень популярен в стартапах.

**YouTrack (JetBrains)** — мощный, гибкий, особенно для разработчиков. Используется внутри JetBrains.

**Yandex Tracker, Kaiten, Pyrus, Bitrix24.** Российские альтернативы. Активно развиваются в РФ как замена Jira.

### Специализированные системы

**Smartsheet.** Хороший выбор для табличных проектов с гибкими структурами.

**Airtable.** Гибкая «база-таблица» с UI, отлично для процессов.

**Targetprocess, Planview, Workfront.** Enterprise PPM, управление портфелями.

**MS Loop.** Совместная работа в реальном времени, как Notion от Microsoft.

### Выбор инструмента

| Сценарий | Что выбрать |
|----------|-------------|
| IT-команда, Scrum | Jira, YouTrack, Linear |
| Небольшая команда, простой Kanban | Trello, Asana, Yandex Tracker |
| Большой план, Гантт | MS Project, Primavera |
| Стройка, инфраструктура | Primavera, Plan-Radar |
| Многоязычная компания, документация | Notion, Confluence |
| Государственные / корпоративные в РФ | Битрикс24, ELMA, Naumen, Kaiten |
| Маркетинг и креативная работа | Wrike, Monday |

### Интеграции

PMIS не работает в изоляции. Стандартные интеграции:

- **Git / GitLab / GitHub** — для разработки. Pull request связаны с задачами.
- **Slack, MS Teams, Telegram** — для уведомлений и обсуждений.
- **BI-системы** (Power BI, Tableau, DataLens) — для отчётности.
- **ERP/HR** — для ресурсов и времени.
- **Time-tracking** (Toggl, Harvest, Hubstaff) — для учёта рабочего времени.
- **Календари** (Google, Outlook) — для встреч и дедлайнов.

### Российские реалии после 2022

После начала ограничений многие компании перешли:

- Jira → **Yandex Tracker**, **Kaiten**, **YouTrack** (Cloud).
- MS Project → **1С-Битрикс24**, **Adept**, **ELMA365**.
- Confluence → **Yandex Wiki**, **Notion** (с осторожностью), **Kaiten**.

### Тренды

- **AI-копилоты.** Помогают планировать, оценивать длительность, суммаризировать статусы. Microsoft Copilot, Jira AI, Notion AI, Linear AI.
- **OKR + проекты.** Связь стратегических целей с конкретными задачами через OKR.
- **Resource AI** — автоматическое распределение задач между членами команды на основе их загрузки и навыков.
- **Async-first.** Статусы и обновления вместо звонков, дайджесты вместо собраний.
- **Process automation** (Zapier, Make, n8n, Power Automate) — связки между инструментами без программирования.

### Метрики, отслеживаемые в PMIS

- **Velocity** (Agile) — сколько story points за спринт.
- **Lead time, cycle time** — время прохождения задачи.
- **Throughput** — пропускная способность.
- **SPI / CPI** — индексы по EVM.
- **Wormhole** — % задач, провисающих в одной колонке Канбан.
- **WIP (Work in Progress)** — сколько одновременно в работе.
- **On-time delivery** — % завершённых в срок.
- **Утилизация ресурсов** — загрузка людей и оборудования.

### Что важно при внедрении PMIS

- **Согласованность процессов** в команде. Инструмент не спасёт, если все работают по-разному.
- **Минимум обязательных полей.** Больше не значит лучше — пользователи устают и саботируют.
- **Шаблоны и автоматизации.** Уменьшают рутину.
- **Регулярная гигиена бэклога.** Удаление неактуального, обновление приоритетов.
- **Обучение и быстрая обратная связь.**
- **Привязка к KPI / OKR** — задачи связаны с целями команды.
- **Интеграции с источниками данных** — без них PMIS становится «островом».

### Что важно сказать в итоге

PMIS — программные системы для управления проектами: планирование, исполнение, контроль, отчётность. Делятся на классические (MS Project, Primavera) для расписаний и Гантт, Agile (Jira, Linear, YouTrack, Yandex Tracker, Kaiten) для гибких команд, корпоративные PPM (Workfront, Planview) для портфельного управления. Современные тренды — AI-копилоты, OKR + проекты, async-first, process automation. Российские альтернативы Jira — Yandex Tracker, Kaiten, YouTrack. Главное при внедрении — согласованность процессов, минимальная бюрократия, интеграции, привязка к OKR/KPI и регулярная гигиена.
