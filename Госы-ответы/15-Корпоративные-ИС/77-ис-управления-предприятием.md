# 77. Информационные системы в управлении предприятием

[← К группе «Корпоративные ИС»](README.md) · [← Ко всем группам](../README.md)

## План ответа

1. Что такое ИС управления предприятием и из чего состоят.
2. Классификация ИС по уровню управления.
3. Основные классы систем (ERP, CRM, SCM, HRM и др.).
4. Архитектура и интеграции.
5. Уровни интеграции и тренды.
6. Безопасность КИС.
7. Российские реалии и импортозамещение.

## Развёрнутый ответ

### Что такое ИС управления предприятием

**Информационные системы управления предприятием** — это комплекс программно-аппаратных решений, обеспечивающих автоматизацию основных и вспомогательных бизнес-процессов организации. Это «нервная система» компании, через которую идут все потоки данных и решений.

Главные задачи:

- автоматизация рутинных операций;
- интеграция данных между подразделениями;
- поддержка принятия управленческих решений;
- обеспечение прозрачности и контроля;
- соответствие регуляторике.

В крупной компании могут работать **сотни** ИС: от ERP до маленьких приложений отдельных команд. Задача EA — обеспечить, чтобы они работали согласованно.

### Классификация ИС по уровню управления

Классическая пирамида ИС:

| Уровень | Назначение | Примеры |
|---------|-----------|---------|
| **Стратегический** | Поддержка решений топ-менеджмента | DSS, EIS, BI, Data Warehouse |
| **Тактический** | Управление функциональными областями | ERP, CRM, SCM, HRM |
| **Операционный** | Учёт повседневных операций | MES, WMS, Kassa, POS, Service Desk |

На каждом уровне свои потребности:

- Топ-менеджмент: панели KPI, прогнозы, сценарии.
- Менеджеры среднего звена: операционные отчёты, бюджеты.
- Сотрудники: транзакционные системы для повседневной работы.

### Основные классы систем

**ERP (Enterprise Resource Planning).** Финансы, производство, склад, кадры, продажи. Сердце КИС.

Глобально: SAP, Oracle, Microsoft Dynamics 365, Infor, Workday.

В России: **1С:ERP, Галактика, Парус, БОСС-Корпорация, Astra ERP**.

**CRM (Customer Relationship Management).** Взаимодействие с клиентами: продажи, маркетинг, поддержка.

Глобально: Salesforce, MS Dynamics 365 Sales, HubSpot, Zoho.

В России: **AmoCRM, Bitrix24 CRM, RetailCRM, Мегаплан**.

**SCM (Supply Chain Management).** Управление цепями поставок.

Глобально: SAP IBP, Oracle SCM Cloud, Blue Yonder, Kinaxis.

В России обычно реализуется как модуль ERP.

**HRM / HCM (Human Capital Management).** Управление персоналом.

Глобально: Workday, SAP SuccessFactors, Oracle HCM, BambooHR.

В России: **1С:ЗУП, Saby, Контур.Зарплата, HRlink**.

**EAM (Enterprise Asset Management).** Управление физическими активами и оборудованием.

Глобально: SAP PM, IBM Maximo, Infor EAM.

В России: **1С:ТОИР, Галактика EAM**.

**PLM (Product Lifecycle Management).** Жизненный цикл изделия — от идеи до утилизации.

Глобально: Siemens Teamcenter, Dassault ENOVIA, PTC Windchill.

В России: **Аскон Лоцман, Т-Flex PLM, Search**.

**BI / Data Warehouse.** Отчётность и аналитика.

Глобально: Power BI, Tableau, Qlik, Looker.

В России: **Yandex DataLens, Visiology, Loginom, FineBI**, open-source Apache Superset.

**DMS / EDMS (Document Management System).** Документооборот.

Глобально: SharePoint, OpenText, Documentum, M-Files.

В России: **Directum, 1С:Документооборот, ЭОС, СБИС**.

**ITSM (IT Service Management).** Управление ИТ-сервисами.

Глобально: ServiceNow, Jira Service Management, BMC Remedy.

В России: **Naumen, SimpleOne, OTRS**.

**MES (Manufacturing Execution System).** Оперативное управление производством.

Глобально: Wonderware, Siemens Opcenter, GE Proficy.

В России: **Цех.MES, Аспэн.MES**.

**WMS (Warehouse Management System).** Управление складом.

Глобально: Manhattan Associates, SAP EWM, Blue Yonder.

В России: **СКЛАД.WMS, Solvo.WMS, AB Solutions**.

**TMS (Transport Management System).** Транспортная логистика.

Глобально: Oracle TMS, Blue Yonder TMS.

В России: **AB-Online, Лардис, MUST**.

**POS / OMS** — для розницы.

### Архитектура КИС

Типичный ИТ-ландшафт крупной компании:

```
[Точки доступа: ПК, мобильные, киоски, POS]
        │
[Веб-приложения / мобильные клиенты]
        │
[API-шлюзы / шины интеграции (ESB, iPaaS, Kafka)]
   ├──── ERP ─── CRM ─── SCM ─── HRM ─── EAM ─── ...
   │     │      │       │       │       │
   └─────────── Master Data Management (MDM) ───
   
[DWH / Lakehouse + BI]
        │
[Аналитика, отчёты, ML-модели]
```

Внизу — операционные системы записи, выше — аналитические системы анализа. MDM (Master Data Management) обеспечивает единый источник правды для ключевых сущностей (клиент, товар, сотрудник).

### Уровни интеграции

Интегрировать системы можно по-разному.

- **Точка-точка** — каждая система с каждой напрямую. На M систем — M² связей, чудовищно при росте.
- **Hub-and-spoke (ESB — Enterprise Service Bus)** — централизованная шина. Все шлют через неё. Снижает сложность, но создаёт single point of failure.
- **iPaaS** (cloud Integration Platform as a Service) — облачная альтернатива. MuleSoft, Workato, Boomi.
- **Event-driven** через Kafka / RabbitMQ / NATS — асинхронные события.
- **GraphQL Federation** на уровне UI — единое API из множества микросервисов.
- **API Gateway + service mesh** для микросервисов — Kong, Istio.

### Безопасность

КИС — лакомая цель для атак. Защита включает:

- **IdM / IGA** — управление учётками и правами.
- **PAM** — привилегированные доступы.
- **SIEM / SOC** — мониторинг событий.
- **DLP** — защита от утечек.
- **Шифрование** данных в покое и в канале.
- **Зональная сегментация сети**.
- **Regulatory compliance** — 152-ФЗ, GDPR, 187-ФЗ (КИИ).

### Российские реалии и импортозамещение

После 2022 года крупные компании были вынуждены массово мигрировать:

- **ERP**: SAP → 1С:ERP, Галактика.
- **CRM**: Salesforce → AmoCRM, Bitrix24.
- **BI**: Tableau → DataLens, Visiology.
- **ITSM**: ServiceNow → Naumen, SimpleOne.
- **DMS**: SharePoint → Directum, 1С:Документооборот, ЭОС.
- **VPN**: Cisco → ViPNet, Континент.
- **Антивирусы**: Symantec → Kaspersky, Dr.Web.

Это огромные миграционные проекты, занимающие 1–3 года и стоящие сотни миллионов рублей.

### Цикл управления

Любая ИС в основе своей реализует цикл управления:

1. **Сбор данных** — транзакции, события.
2. **Хранение** — ERP, БД, DWH, lake.
3. **Обработка** — бизнес-логика, аналитика.
4. **Интерпретация** — дашборды, отчёты.
5. **Решения** — пользовательский интерфейс, авто-сценарии.
6. **Действия** в физическом мире.

### Управленческое решение и DSS

**DSS (Decision Support System)** помогает принимать решения через:

- **сценарии «что если»**;
- **модели оптимизации**;
- **статистический анализ**;
- **интерактивные визуализации**;
- **встроенные ML-модели**.

В современных ERP-системах DSS интегрирован: вместо отдельной системы — встроенная аналитика и AI-копилоты прямо в интерфейсе работы.

### Тренды

- **Cloud-first архитектура** — облако по умолчанию.
- **Composable enterprise** — модульные сервисы вместо монолитных систем.
- **AI-копилоты в каждой системе** — Microsoft Copilot, SAP Joule, Salesforce Einstein.
- **Self-service BI** — бизнес-пользователи строят отчёты сами, без аналитиков.
- **Data products** — данные как продукт со своим владельцем и SLA.
- **Process mining** — оптимизация процессов через анализ логов.
- **Низкокодовые / no-code платформы** — Mendix, OutSystems, ELMA365, BPMSoft.
- **Sustainability reporting (ESG)** — обязательная отчётность по экологии и социальной ответственности.
- **Cybersecurity-by-design** — безопасность с самого начала.

### Что важно при выборе

При выборе ИС для предприятия учитывают:

- **соответствие бизнес-процессам** — насколько система ложится на нашу специфику;
- **гибкость и API** — можно ли интегрировать;
- **стоимость владения (TCO)** — не только цена лицензий, но и эксплуатация;
- **регуляторика и сертификация** — для гос/медицины/банков;
- **импортонезависимость** — в РФ актуально;
- **сообщество и поддержка** — есть ли консультанты, документация;
- **roadmap вендора** — куда движется продукт.

### Что важно сказать в итоге

Информационные системы управления предприятием — это комплекс программных решений для автоматизации бизнес-процессов. По уровню управления делятся на стратегические (DSS, BI, EIS), тактические (ERP, CRM, SCM, HRM, EAM, PLM), операционные (MES, WMS, TMS, POS, Service Desk). Центром обычно является ERP, вокруг — специализированные системы. Архитектура многоуровневая: точки доступа → приложения → шины интеграции → DWH/BI. Интеграция строится через ESB, iPaaS, Kafka. Безопасность — IdM, PAM, SIEM, DLP, шифрование. После 2022 в РФ идёт активное импортозамещение: SAP → 1С, ServiceNow → Naumen, Tableau → DataLens. Тренды — cloud-first, composable enterprise, AI-копилоты, self-service BI, low-code платформы.
