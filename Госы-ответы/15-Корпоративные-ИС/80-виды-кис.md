# 80. Виды и примеры корпоративных информационных систем

[← К группе «Корпоративные ИС»](README.md) · [← Ко всем группам](../README.md)

## План ответа

1. Принципы классификации КИС.
2. Классификация по функциональному назначению.
3. По уровню управления (TPS, MIS, DSS, EIS).
4. По способу разворачивания (on-prem, cloud, hybrid).
5. По архитектуре (монолит, SOA, микросервисы).
6. По отраслям.
7. Современный российский ландшафт.

## Развёрнутый ответ

### Принципы классификации

КИС — это огромное семейство систем, и единой «лучшей» классификации нет. Их группируют по разным основаниям:

- **функциональному назначению** — что делает система;
- **уровню управления** — кто пользуется;
- **способу разворачивания** — где размещается;
- **архитектуре** — как устроена внутри;
- **отрасли** — для какого бизнеса.

Эти классификации не взаимоисключающие — одна и та же система попадает в несколько категорий.

### По функциональному назначению

Это самый практичный взгляд: «какой класс системы мне нужен?»

**ERP (Enterprise Resource Planning).** Управление ресурсами предприятия.

- Глобальные: SAP S/4HANA, Oracle Fusion, Microsoft Dynamics 365, Infor, Workday.
- Российские: **1С:ERP, Галактика, Парус, БОСС-Корпорация, Astra ERP**.

**CRM (Customer Relationship Management).** Взаимодействие с клиентами.

- Глобальные: Salesforce, HubSpot, MS Dynamics 365 Sales, Zoho CRM.
- Российские: **AmoCRM, Bitrix24 CRM, RetailCRM, Мегаплан**.

**SCM (Supply Chain Management).** Цепи поставок.

- Глобальные: SAP IBP, Oracle SCM Cloud, Blue Yonder, Kinaxis.
- В России — часто как модуль ERP.

**HRM / HCM.** Управление персоналом.

- Глобальные: Workday, SAP SuccessFactors, Oracle HCM, BambooHR.
- Российские: **1С:ЗУП, Saby, Контур.Зарплата, HRlink**.

**BI / Analytics.**

- Глобальные: Power BI, Tableau, Qlik, Looker.
- Российские: **Yandex DataLens, Visiology, Loginom, FineBI**, open-source Apache Superset.

**EAM (Enterprise Asset Management).** Управление активами и ТОиР.

- Глобальные: SAP PM, IBM Maximo, Infor EAM.
- Российские: **1С:ТОИР, Галактика EAM**.

**PLM (Product Lifecycle Management).** Жизненный цикл изделия.

- Глобальные: Siemens Teamcenter, Dassault ENOVIA, PTC Windchill.
- Российские: **Аскон Лоцман, Т-Flex PLM, Search**.

**MES (Manufacturing Execution System).** Оперативное управление производством.

- Глобальные: Siemens Opcenter, GE Proficy, Wonderware.
- Российские: **Цех.MES, Аспэн.MES**.

**WMS, TMS, OMS.** Логистика — подробно в вопросе 78.

**DMS / EDMS.** Документооборот.

- Глобальные: SharePoint, OpenText, Documentum, M-Files.
- Российские: **Directum, 1С:Документооборот, ЭОС, СБИС**.

**ITSM (IT Service Management).**

- Глобальные: ServiceNow, Jira Service Management, BMC Remedy.
- Российские: **Naumen, SimpleOne, OTRS**.

Этот список можно продолжать: SRM (поставщики), PIM (product information), MDM (master data), GRC, RPA (роботизация процессов)... Современная корпорация имеет десятки специализированных систем.

### По уровню управления

Это пирамида ИС по тому, кому они служат.

**Transaction Processing System (TPS).** Учёт ежедневных операций — кассовые системы, банковские транзакции, обработка заказов. Нижний уровень, на котором происходят реальные действия.

**Management Information System (MIS).** Отчёты и сводки для среднего менеджмента — что произошло за день/неделю/месяц.

**Decision Support System (DSS).** Поддержка принятия решений — сценарии «что-если», оптимизационные модели.

**Executive Information / Support System (EIS / ESS).** Для топ-менеджмента — стратегические дашборды, KPI верхнего уровня.

**Knowledge Management System (KMS).** Управление знаниями — базы, wiki, корпоративные академии.

**Expert System.** Узкоспециализированная экспертная — медицинская диагностика, экспертиза ущерба.

### По способу разворачивания

Где живёт система?

| Тип | Где | Примеры |
|-----|-----|---------|
| **On-premise** | На своих серверах | SAP ECC, 1С:ERP |
| **Private cloud** | Свой облачный кластер | Astra Linux Cloud, VK Cloud Enterprise |
| **Public SaaS** | Облако вендора | Salesforce, Workday, NetSuite |
| **Hybrid** | Часть on-prem, часть в облаке | Большие корпорации |
| **Edge / IIoT** | На объектах в реальном мире | Industry 4.0 |

В 2020-х наблюдается массовый сдвиг в облако: дешевле эксплуатировать, быстрее внедрять, проще масштабировать. Но крупные госорганизации и КИИ часто остаются on-prem из соображений безопасности.

### По архитектуре

**Монолит.** Одна большая инсталляция. Все модули в одном процессе или плотно связаны. Классический SAP R/3, традиционный 1С:Предприятие. Плюсы — простота развертывания; минусы — сложно масштабировать, любое изменение затрагивает всё.

**SOA (Service-Oriented Architecture).** Сервисы общаются через **ESB**. Появилась в 2000-х. Сейчас уступает микросервисам.

**Микросервисы.** Каждый сервис независим, общается через API или события. Развёртывание в Kubernetes. Гибко, но сложнее в эксплуатации.

**Composable enterprise.** Собрано из модулей разных вендоров через iPaaS. Будущее enterprise-архитектуры.

### По отрасли

Многие КИС адаптированы под специфику отрасли.

- **Retail.** POS, OMS, loyalty programs, e-commerce, retail-аналитика.
- **Manufacturing.** MES, MRP II, PLM, EAM.
- **Banking.** Core banking (CFT, Diasoft, ЦФТ), PFM, anti-fraud, кредитные системы.
- **Insurance.** Policy management, claims (Guidewire, ИБС от 1С).
- **Healthcare.** HIS (Hospital Information System), ЭМК (электронная медкарта), лабораторные.
- **Logistics / Transport.** TMS, fleet management, маршрутизация.
- **Telecom.** OSS / BSS (Operations / Business Support Systems).
- **Education.** LMS (Moodle, Stepik), СЭО (1С:Образование, ELiS).
- **Государственный сектор.** ГИИС, ГИС, Госуслуги, ведомственные системы.

Под каждую отрасль есть как отечественные, так и зарубежные решения.

### Современный российский ландшафт

После 2022 года резкая смена приоритетов: импортозамещение стало обязательным для всех стратегических отраслей.

| Класс | Было | Стало (РФ) |
|-------|------|------------|
| ERP | SAP, Oracle | 1С:ERP, Галактика |
| CRM | Salesforce | AmoCRM, Bitrix24 |
| BI | Tableau, Power BI | DataLens, Visiology |
| ITSM | ServiceNow | Naumen, SimpleOne |
| DMS | SharePoint | Directum, 1С:ДО, ЭОС |
| VPN | Cisco | ViPNet, Континент |
| Антивирус | Symantec | Kaspersky, Dr.Web |
| Облако | AWS, Azure | Яндекс Cloud, VK Cloud |
| База данных | Oracle | PostgreSQL, Postgres Pro |

В реестре отечественного ПО Минцифры — тысячи продуктов. Для критической инфраструктуры (КИИ) использование иностранного ПО с 2025 года практически запрещено.

### Тренды

- **AI-копилоты в каждой системе.** SAP Joule, Microsoft Copilot, Salesforce Einstein, Yandex GPT в Tracker.
- **Embedded analytics.** Отчёты прямо внутри ERP/CRM, не нужен отдельный BI-инструмент.
- **Composable enterprise.** Модульная архитектура.
- **Cloud-native.** Kubernetes, контейнеры, multi-cloud.
- **Data Mesh.** Данные как продукты доменов.
- **Process Mining.** Оптимизация процессов через анализ логов.
- **No-code / low-code платформы.** Mendix, OutSystems, ELMA365, BPMSoft в России.

### Выбор системы

При выборе КИС учитывают:

- **Соответствие бизнес-процессам.** Насколько система ложится на нашу специфику.
- **Гибкость и API.** Можно ли интегрировать с другими системами.
- **Стоимость владения (TCO).** Не только лицензия, но и эксплуатация, обновления, кастомизация.
- **Сертификация / совместимость с регуляторикой** для гос/медицины/банков.
- **Импортонезависимость.** В РФ — критично для стратегических отраслей.
- **Сообщество и поддержка.** Есть ли консультанты, документация, форумы.
- **Roadmap вендора.** Куда движется продукт.

### Что важно для успеха

После выбора и внедрения важно:

- **Чёткое понимание бизнес-целей** — зачем эта система.
- **Регулярная архитектурная ревизия** (раз в 1–2 года).
- **Управление мастер-данными (MDM)** — без качества данных любая система работает плохо.
- **Безопасность и комплаенс.**
- **Адаптация культуры** к работе с системой.

### Что важно сказать в итоге

КИС классифицируются по нескольким основаниям: по **функциональному назначению** (ERP, CRM, SCM, HRM, BI, EAM, PLM, MES, WMS, DMS, ITSM), по **уровню управления** (TPS, MIS, DSS, EIS, KMS), по **способу разворачивания** (on-prem, private cloud, public SaaS, hybrid, edge), по **архитектуре** (монолит, SOA, микросервисы, composable), по **отрасли** (retail, manufacturing, banking, healthcare, telecom, education, govtech). В РФ после 2022 года активное импортозамещение: SAP → 1С, Salesforce → Bitrix24, Tableau → DataLens, ServiceNow → Naumen. Тренды — AI-копилоты, embedded analytics, composable enterprise, cloud-native, data mesh, low-code платформы. Выбор системы — это компромисс между функциональностью, гибкостью, TCO, регуляторикой и импортонезависимостью.
