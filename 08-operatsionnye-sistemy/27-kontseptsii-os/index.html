Развёрнутый ответ

Что такое ОС

Операционная система (ОС) — это базовое системное программное обеспечение, которое управляет ресурсами компьютера и предоставляет интерфейс между пользовательскими программами и аппаратурой. Без ОС каждая программа должна была бы сама знать, как работать с диском, сетью, экраном, клавиатурой — что нереально.

ОС решает две главные задачи:

  1. Управление ресурсами — процессорным временем, памятью, дисками, сетью, устройствами. Если десятки процессов хотят одновременно работать с диском, ОС регулирует, кто и когда получит доступ.
  2. Предоставление абстракций — программист не работает с физическими секторами диска, он работает с файлами. Не с физическими страницами памяти, а с непрерывным виртуальным адресным пространством. Эти абстракции и есть «удобство», за которое мы любим современные ОС.

Основные функции ОС

  1. Управление процессами — создание, уничтожение, планирование, синхронизация, межпроцессное взаимодействие.
  2. Управление памятью — виртуальная память, защита, страничная организация, swap.
  3. Управление файлами и устройствами — файловые системы, драйверы, буферизация I/O.
  4. Управление пользователями и безопасностью — учётные записи, аутентификация, права доступа.
  5. Сетевые сервисы — стек TCP/IP, сокеты, маршрутизация.
  6. Виртуализация — VM, контейнеры, гипервизоры (KVM, Hyper-V).

Архитектуры ядра

Главное архитектурное решение — что и где запускается.

Монолитное ядро. Все сервисы (управление процессами, памятью, ФС, сетью) в одном бинарнике в режиме ядра. Быстро, потому что нет накладных расходов на переключения. Уязвимо к ошибкам — баг в драйвере роняет всю систему. Примеры: Linux, классический UNIX, FreeBSD.

Микроядро. В режиме ядра — минимум: планировщик, IPC, базовое управление памятью. Всё остальное (драйверы, ФС, сеть) — отдельные процессы в user mode. Стабильнее, безопаснее, но медленнее из-за частых переключений. Примеры: MINIX 3, QNX, L4, sel4.

Гибридное ядро. Компромисс: часть сервисов в ядре, часть — вне. Примеры: Windows NT (и потомки — Windows 11), macOS (XNU = Mach + BSD), Haiku.

Экзоядро. Ядро лишь распределяет ресурсы, прикладные «библиотечные ОС» сами реализуют абстракции. Экспериментальное направление, MIT Exokernel.

Наноядро. Ещё меньше микроядра — только базовое управление прерываниями.

Linux выбирает монолитную архитектуру с модулями: основной функционал в одном бинарнике, но драйверы можно подгружать на лету (insmod, modprobe). Это даёт компромисс — производительность монолита + гибкость микроядра.

Процессы и потоки

Процесс — это экземпляр выполняемой программы со своей памятью, файловыми дескрипторами, окружением. У процесса есть PID, состояние, родитель, дочерние процессы.

Поток (thread) — единица планирования внутри процесса. Потоки одного процесса разделяют память (heap, глобальные переменные, файлы), но имеют свой стек и регистры.

Состояния процесса (классическая схема):

new → ready → running → waiting → ready → ... → terminated

ОС поддерживает каждый процесс структурой PCB (Process Control Block), где хранятся PID, состояние, регистры, контекст, открытые файлы, права.

Планировщик

Планировщик решает, какой готовый процесс/поток получит CPU. Алгоритмы:

  • FCFS (First-Come-First-Served) — простая очередь.
  • SJF (Shortest Job First) — короткие задачи первыми, оптимально по среднему ожиданию.
  • Round Robin — каждому квант времени, основа интерактивных систем.
  • Priority Scheduling — с приоритетами, опасен голоданием.
  • MLFQ (Multi-Level Feedback Queue) — много очередей с разными приоритетами, динамическое перемещение.
  • CFS (Completely Fair Scheduler) в Linux до 6.6 — справедливая разделка CPU.
  • EEVDF в Linux с 6.6 — улучшенная версия CFS.

Современные планировщики учитывают NUMA, кэш-локальность, тепловой режим CPU, энергопотребление.

Управление памятью

Виртуальная память. Каждому процессу даётся своё непрерывное адресное пространство, не зависящее от физической памяти. Это обеспечивает изоляцию и удобство.

Страничная организация. Память делится на страницы по 4 КБ (или больше для huge pages). Виртуальные страницы отображаются в физические через таблицы страниц, поддерживаемые MMU аппаратно. Для ускорения используется TLB — кэш переводов.

Swap. Если RAM заполнена, ОС вытесняет неиспользуемые страницы на диск. Подробно — в вопросе 40.

Copy-on-write. При fork() родитель и потомок разделяют страницы; копирование происходит только при записи. Это экономит память.

OOM-killer в Linux — убивает процесс, когда память кончилась, по эвристике «кого можно».

Ввод-вывод

Прерывания (IRQ) — устройство сигнализирует CPU, и тот переключается на обработчик. Альтернатива — polling (опрос), но он расходует CPU зря.

DMA (Direct Memory Access) — устройство пишет в память напрямую, без участия CPU. Так работают сетевые карты, диски.

Memory-mapped I/O — регистры устройств видны как обычные адреса памяти.

Драйверы реализуют конкретное «общение» с устройством. Могут быть в ядре или, как FUSE, в user space.

Современный асинхронный I/O в Linux — io_uring (с 5.1), обеспечивает высокую производительность без блокировок.

Системные вызовы

Системный вызов (syscall) — точка входа из user mode в kernel mode. Прикладная программа вызывает read, open, write, mmap — это всё syscall-обёртки.

ssize_t n = read(fd, buf, sizeof(buf));

Под капотом — специальная инструкция CPU (syscall на x86-64), которая сохраняет контекст и переключается в ядро. Обратное переключение — после выполнения операции.

Linux дополнительно использует vDSO — быстрые syscalls без перехода в ядро (например, gettimeofday).

Виды ОС

По назначению:

  • Пакетные — исторически, обрабатывали задания пакетами без интерактивности.
  • Разделения времени — UNIX и потомки, многопользовательский режим.
  • Реального времени (RTOS) — FreeRTOS, VxWorks, QNX. Гарантируют время отклика, используются в авионике, медицине, промышленной автоматике.
  • Сетевые / распределённые — Plan 9, Amoeba (исторически).
  • Мобильные — Android (на ядре Linux), iOS (на ядре XNU).
  • Встраиваемые — Embedded Linux, Zephyr, RIOT.
  • Серверные — оптимизированы под нагрузку, минимум GUI (Linux RHEL/Ubuntu Server, Windows Server).
  • Десктопные — оптимизированы под пользовательский интерфейс.

Что важно сказать в итоге

Операционная система — это посредник между приложениями и железом, выполняющий пять главных функций: управление процессами, памятью, файлами и устройствами, пользователями и безопасностью, сетью. Архитектуры ядра — монолитное (Linux), микроядро (QNX), гибридное (Windows NT, macOS), экспериментальные экзо- и наноядра. Базовые понятия — процесс, поток, планировщик, виртуальная память, syscalls — это фундамент любого современного ПО. По назначению ОС делятся на пакетные, разделения времени, реального времени, мобильные, встраиваемые, серверные и десктопные.
